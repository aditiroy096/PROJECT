Research Paper

Reliable Real Time Wireless Monitoring, Automation and Fault Detection System

Prof. Sandip Kale  |  Tarun Zaliya  |  Pratyaksh Ganvir  |  Dipak Khanorkar  |  Aniket Jadhav  |  Samiksha Dhamdar

Department of Electrical Engineering, Jhulelal Institute of Technology, Nagpur, Maharashtra, India

Abstract — Recent advancements in the Internet of Things (IoT) have enabled the development of intelligent monitoring systems capable of collecting and transmitting data in real time. This paper presents the design and implementation of a wireless monitoring, automation, and fault detection system based on the ESP8266 Wi-Fi microcontroller. The proposed system integrates multiple sensors such as a DHT11 temperature and humidity sensor, MQ2 gas sensor, PIR motion sensor, and flame sensor to monitor environmental conditions and detect potential hazards. Sensor data is processed by the ESP8266 and transmitted wirelessly to a cloud server where it can be accessed through a web interface using mobile devices or computers. The system also incorporates a relay module that enables remote control of electrical appliances. The monitoring system can operate both with internet connectivity through cloud services and without internet using direct Wi-Fi communication. Experimental implementation shows that the system is reliable, economical, and suitable for real-time safety monitoring and automation applications.

I. Introduction

The continuous growth of digital technologies has significantly transformed the way monitoring and control systems are designed. In recent years, the Internet of Things (IoT) has emerged as a powerful technology that enables communication between physical devices through wireless networks. IoT-based systems allow sensors and embedded devices to collect environmental data and transmit it to remote servers for analysis and monitoring.

Conventional monitoring systems often rely on manual supervision and lack the ability to provide immediate alerts during emergency situations such as fire accidents, gas leakage, or unauthorized movement. The integration of IoT technology with environmental sensors offers an efficient solution for real-time monitoring and automation.

The ESP8266 microcontroller has gained popularity in IoT applications because it combines processing capability with integrated Wi-Fi connectivity at a low cost. The system proposed in this research focuses on the development of a real-time wireless monitoring and automation platform that integrates multiple sensors with an ESP8266 microcontroller.

II. Review of Literature

Several researchers have proposed IoT-based monitoring systems to improve energy management, environmental safety, and automation. Santos and Ferreira developed an IoT-based monitoring system for real-time tracking of electrical power consumption. Josué et al. designed a home energy monitoring prototype using wireless communication. Kim et al. implemented a ZigBee-based monitoring system for energy management. Balamurugan and Saravanakamalam proposed an IoT-based energy management system using internet connectivity.

Based on these studies, it is clear that there is a need for a low-cost and flexible monitoring system that integrates environmental sensing, automation, and wireless communication. The proposed system addresses this requirement by combining several sensors with the ESP8266 Wi-Fi microcontroller and cloud-based data monitoring.

III. Methodology

The developed monitoring system consists of several hardware components connected to the ESP8266 microcontroller:

  1. ESP8266 Wi-Fi Microcontroller — Main controller with integrated Wi-Fi for wireless communication. Collects sensor data, processes information, and transmits to cloud server.
  2. DHT11 Temperature & Humidity Sensor — Measures environmental temperature and relative humidity. Provides digital output directly readable by ESP8266.
  3. MQ2 Gas & Smoke Sensor — Detects combustible gases such as LPG, methane, propane, and smoke. Works based on change in electrical resistance when exposed to different gases.
  4. PIR Motion Sensor — Detects motion by sensing changes in infrared radiation. Useful for security monitoring systems.
  5. Flame Sensor — Detects fire by sensing infrared light emitted from flames. Triggers alert when flame radiation is detected within range.
  6. Relay Module — Acts as electrically operated switch for controlling high-voltage appliances. Provides electrical isolation between control circuit and load circuit.
  7. Firebase Cloud Platform — Stores and manages sensor data. Enables real-time data synchronization between hardware and web interface.

IV. Working

The system continuously collects environmental data from various sensors connected to the ESP8266. The microcontroller processes the incoming sensor data and compares it with predefined threshold values. If any abnormal condition is detected such as rise in temperature, gas leakage, fire detection, or motion in a restricted area, the system immediately generates an alert.

The system can function in two modes: Offline Mode — direct connection between ESP8266 and user device through Wi-Fi; and Online Mode — data transmission through cloud services for remote monitoring.

V. Results and Discussion

The developed system was tested under various environmental conditions to evaluate its performance. The sensors successfully detected temperature variations, humidity changes, smoke presence, and motion in the monitored environment. The ESP8266 microcontroller effectively processed the sensor data and transmitted the information to the cloud platform. The relay module enabled remote switching of electrical appliances, demonstrating the automation capability of the system.

VI. Conclusion

This research presented the design and implementation of a real-time wireless monitoring, automation, and fault detection system using the ESP8266 microcontroller. The system integrates multiple environmental sensors to monitor temperature, humidity, gas leakage, motion, and fire conditions. The proposed system offers several advantages including low cost, real-time monitoring capability, remote accessibility, and improved safety. Future improvements may include integrating additional sensors, enhancing mobile application interfaces, and implementing advanced data analytics for predictive fault detection.

🔧 Fault Detection & Troubleshooting Guide

🔥 FAULT 1: Fire Alert Triggered
  • 1Check the monitored area immediately for any visible fire or smoke source.
  • 2Press "STOP BUZZER" on the dashboard to silence the alarm.
  • 3All electrical loads are automatically turned OFF by the system for safety — verify this on the dashboard.
  • 4If fire is confirmed, evacuate the area and contact emergency services immediately.
  • 5If it is a false alarm (candle, lighter, etc.), remove the heat source from sensor range.
  • 6Once area is clear, use the Safety Switch to re-enable load control on the dashboard.
  • 7Clean the flame sensor lens with a dry cloth if false triggers occur frequently.
💨 FAULT 2: High Smoke / Gas Alert
  • 1Immediately ventilate the area — open windows and doors.
  • 2Identify the source of smoke or gas (cooking, LPG leak, burning material).
  • 3Turn off LPG/gas cylinders or cooking appliances at the source.
  • 4Press "STOP BUZZER" on dashboard once area is safe.
  • 5Check smoke threshold in Settings — increase it if cooking smoke causes frequent false alarms.
  • 6Allow the MQ2 sensor to breathe fresh air for 5 minutes before system resets to normal.
  • 7If gas leak is suspected, do NOT operate any electrical switches — leave the building and call gas agency.
🌡️ FAULT 3: Overtemperature Alert
  • 1Check the monitored area for any overheating equipment or open heat sources near the sensor.
  • 2Turn off any heating appliances in the area.
  • 3Improve ventilation or air conditioning in the monitored space.
  • 4Press "STOP BUZZER" once temperature starts dropping on the dashboard.
  • 5If temperature reading seems incorrect, check DHT11 sensor connections (VCC, GND, DATA).
  • 6Adjust temperature threshold in Settings (⚙️) if the environment naturally runs warm.
📡 FAULT 4: Dashboard Shows "Disconnected"
  • 1Check that the ESP8266 is powered ON (LED should be glowing).
  • 2Verify that your device (phone/laptop) is connected to the internet.
  • 3Check that the home WiFi router is working and ESP8266 is within range.
  • 4Open Serial Monitor (115200 baud) — check if ESP8266 shows "WiFi CONNECTED".
  • 5If WiFi connection failed, re-upload the code with correct SSID and password.
  • 6Check Firebase console — if database rules are set to private, update them to allow read/write.
  • 7Try power cycling the ESP8266 (unplug and replug power).
🔌 FAULT 5: Relay / Load Not Responding to Toggle
  • 1Check if Safety Switch is ACTIVE on dashboard — loads are locked when safety is ON.
  • 2Check if any alert (Fire/Temp/Smoke) is active — all loads auto-disable during alerts.
  • 3Verify relay module VCC is connected to 5V (not 3.3V).
  • 4Check that relay GND is common with ESP8266 GND.
  • 5Verify IN1-IN4 pins of relay match D5-D8 on ESP8266.
  • 6Check relay LED indicator — it should light up when toggle is ON.
  • 7Use a multimeter to measure voltage at relay output terminals (NO and COM).
🌀 FAULT 6: Sensor Showing Wrong / Zero Values
  • 1Check DHT11 connections — VCC to 3.3V, GND to GND, DATA to D4.
  • 2Open Serial Monitor — if temperature shows 0.0°C, DHT11 is not responding.
  • 3Try replacing DHT11 sensor — they can fail due to static discharge.
  • 4For MQ2 sensor, allow 2-3 minutes warmup time after power ON before readings stabilize.
  • 5Check A0 pin connection for smoke sensor analog output.
  • 6Ensure no two sensors share the same pin — verify pin assignments in the code.
🔄 FAULT 7: Mode Switch Not Working (Online ↔ Offline)
  • 1For Online → Offline: Ensure Firebase is connected (status dot must be green).
  • 2For Offline → Online: Ensure you are connected to "GROUP 1 PROJECT" hotspot and dashboard is open.
  • 3Wait 10-15 seconds after pressing switch button for ESP8266 to restart.
  • 4If ESP8266 fails to connect to home WiFi, it will automatically fall back to Offline Mode.
  • 5Verify home WiFi credentials (SSID and password) in the ESP8266 code are correct.
  • 6Re-upload ESP8266 code if mode switching is repeatedly failing.
